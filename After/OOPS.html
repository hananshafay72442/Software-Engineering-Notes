Introduction to Java

Lecture 7A

What is Java · Features · Platform Components · Applications · IDEs

History
Origin of Java
  • Developed at Sun Microsystems in 1991 by James Gosling
  • Original name: Oak, part of the "Green" project — aimed at consumer electronics
  • Officially released as Java 1.0 in 1996
  • Goals: small, fast, efficient, portable — same goals that made it perfect for the internet
  • Now owned by Oracle after acquiring Sun in 2010
Core Concept
What is Java?
  • General-purpose, class-based, object-oriented programming language
  • Also called the Internet Programming Language
  • Principle: "Write Once, Run Anywhere" (WORA)
  • Source code compiles to platform-independent bytecode (.class files)
  • Bytecode runs on any machine that has a JVM installed
Paradigms
Imperative vs Declarative
  • Imperative — how to do it, step by step (traditional Java loops)
  • Declarative — what to do, let the system figure out how (Java Streams API)
  • Java supports both styles
Imperative
// Find even numbers, multiply by 2 (imperative)
for(int i=0; i<arr.length; i++) {
  if(arr[i] % 2 == 0)
    result.add(arr[i] * 2);
}
// Declarative (Java Streams)
Arrays.stream(arr).filter(n -> n%2==0)
      .map(n -> n*2).collect(toList());
Features (1–4)
  • Simple — modeled on C++ but removes pointers, manual memory; auto GC
  • Object-Oriented — designed OO from the start; supports encapsulation, inheritance, polymorphism
  • Distributed — networking built-in; supports RMI and CORBA for remote communication
  • Interpreted — bytecode interpreted by JVM; platform-neutral .class files
Features (5–8)
  • Robust — strong type checking, exception handling, no error-prone pointer constructs
  • Secure — null checking, type casting verification, array bounds checking, Security Manager
  • Architecture-Neutral — bytecode is OS-independent; same .class runs anywhere
  • Portable — architecture-neutral → runs on any platform without recompilation
Features (9–11)
  • High Performance — bytecode includes compile info; JIT compiler speeds execution
  • Multithreaded — parallel tasks built-in; smoothly integrated thread API
  • Dynamic — new code can be loaded on-the-fly without recompilation; extended class libraries; multiple inheritance via interfaces
Platform Components
JDK · JRE · JVM
  • JDK (Java Development Kit) — for developers. Contains: javac compiler, java launcher, appletviewer, javadoc, debugger + JRE
  • JRE (Java Runtime Environment) — for users. Contains: JVM + standard class libraries. No development tools
  • JVM (Java Virtual Machine) — executes bytecode; platform-specific but provides platform-independent environment to programs
  • Relationship: JDK ⊃ JRE ⊃ JVM
Editions
Java SE · EE · ME
  • Java SE (Standard Edition) — desktop/standalone apps and applets; most common
  • Java EE (Enterprise Edition) — server-side apps; Servlets, JSP, Enterprise JavaBeans
  • Java ME (Micro Edition) — mobile and embedded devices, cell phones
Application Types
Types of Java Apps
  • Desktop Applications — run on local computer, based on J2SE
  • Java Applets — run inside a web browser, J2SE
  • Java Servlets — run on application server, J2EE
  • Enterprise JavaBeans (EJB) — server-side business logic, J2EE
  • Android Apps — mobile development using Java
Compile & Run — Step by Step
  • Step 1: Write source code in .java file (filename = class name)
  • Step 2: Compile with javac HelloWorld.java → produces HelloWorld.class (bytecode)
  • Step 3: Run with java HelloWorld → JVM interprets bytecode
  • C++ → compiles directly to platform-specific machine code. Java → compiles to platform-neutral bytecode first
HelloWorld.java
public class HelloWorld {
  public static void main(String[] args) {
    System.out.println("Hello, World!");
  }
}
// Terminal:
// javac HelloWorld.java   → creates HelloWorld.class
// java HelloWorld         → Hello, World!
IDEs & Language Spec
  • IDE (Integrated Development Environment) — editor + compiler + debugger in one GUI
  • Popular Java IDEs: Eclipse, IntelliJ IDEA, NetBeans, JCreator, JBuilder, Oracle JDeveloper, TextPad
  • Java Language Specification — technical definition of Java syntax and semantics at java.sun.com
  • Java API — predefined classes and interfaces for development; always expanding
  • API docs: docs.oracle.com/javase/8/docs

Interpreter & JVM Internals

Lecture 7B

JDK · JRE · JVM · JIT · PATH · CLASSPATH · Execution Flow

JDK (Java Development Kit)
  • Contains tools required to develop Java programs
  • Tools: javac.exe (compiler), java.exe (launcher), appletviewer, javadoc
  • .java file = Java source code (human-readable) → compiled by javac
  • Output of compilation = .class file containing bytecode
  • JDK includes the JRE (so you can also run programs)
JRE (Java Runtime Environment)
  • Contains JVM + class libraries + supporting files
  • No development tools — no compiler, no debugger
  • Used to run already-compiled Java programs
  • JVM becomes an instance of JRE at runtime — also called a runtime interpreter
JVM
Java Virtual Machine
  • An abstract computing machine — not physical hardware
  • Platform-specific implementation but provides platform-independent environment
  • Translates bytecode to machine code for the underlying OS + hardware
  • Responsibilities: garbage collection, array bounds checking, type checking
  • Called "virtual" because it doesn't depend on actual OS/hardware architecture
  • .class bytecode file → understood and run by JVM
JIT
Just-In-Time Compiler
  • Part of the JVM; used to speed up execution time
  • Dynamic translation — compilation happens at runtime, not before
  • Translates bytecode → native machine code just before it is executed
  • Compiles parts of bytecode with similar functionality at the same time
  • Reduces total compilation time compared to interpreting every instruction
  • Translator: JVM instruction set → specific CPU instruction set
PATH vs CLASSPATH
  • PATH = environment variable used by the OS to find executable programs (e.g. javac, java)
  • CLASSPATH = environment variable used by Java to find compiled .class files, JAR files, ZIP files
  • Short rule: PATH = only for executables  |  CLASSPATH = for .class, .jar, .zip
Command Prompt (Windows)
// Set PATH to JDK bin folder
set path=C:\Program Files\Java\jdk-22\bin

// Set CLASSPATH to current directory
set classpath=.

// Compile
javac Welcome.java

// Run
java Welcome
Java Execution Flow (Step by Step)
  • Step 1: Write Welcome.java — human-readable Java source code
  • Step 2: javac Welcome.java — compiler produces Welcome.class (bytecode)
  • Step 3: JVM loads the .class file via the Class Loader
  • Step 4: Bytecode Verifier checks the bytecode for security/validity
  • Step 5: JVM Interpreter executes bytecode instruction by instruction
  • Step 6: JIT compiler speeds up frequently executed code
Flow
Welcome.java ──[javac]──▶ Welcome.class ──[JVM]──▶ Output
  (source)                  (bytecode)      Interpreter + JIT
First Java Program — Traced
Welcome.java

  public static void main(String[] args) {
    System.out.println("Welcome to Java!");
  }
}
Trace Explanation
  • public class Welcome — defines a class; filename must be Welcome.java
  • public static void main(String[] args) — entry point; JVM calls this to start
  • System.out.println(...) — calls the println method on the standard output stream
  • import java.lang.* — automatically imported; not needed to write explicitly
  • All Java classes implicitly extend Object (the root of the class hierarchy)

Java Elementary Programming I

Lecture 8A

Imports · Comments · Keywords · Modifiers · Classes · main() · Statements

Packages & Import
Importing Packages
  • java.lang.* is automatically imported — no need to write it explicitly
  • All other packages need an explicit import statement
  • Example: import java.util.Scanner; for Scanner class
  • import java.util.*; imports all classes in java.util
Comments
Three Types of Comments
  • Line comment: preceded by // — ignored until end of line
  • Paragraph comment: enclosed in /* ... */ — can span multiple lines
  • Documentation comment: /** ... */ — used with javadoc to auto-generate HTML docs
  • @tags in doc comments: @author, @param, @return etc.
  • Run javadoc Test.java or javadoc *.java to generate docs
Reserved Words
Keywords
  • Keywords have specific meaning to the compiler and cannot be used as identifiers
  • Examples: class, public, static, void, int, for, if, new, return, this, extends, implements
  • true, false, null are literals — also cannot be used as identifiers
Modifiers
Access & Non-Access Modifiers
  • Modifiers specify properties and behavior of data, methods, and classes
  • Access modifiers: public, private, protected
  • Non-access modifiers: static, final, abstract
  • public — accessible by any program
  • private — accessible only within declaring class
Classes
The Class — Essential Construct
  • A class is a template for objects — the essential building block of Java
  • A class contains: data fields (state) + methods (behavior) + constructors
  • Every Java program must have at least one class
  • File name must match the public class name exactly
main Method
Program Entry Point
  • JVM starts execution by calling main()
  • Must be exactly: public static void main(String[] args)
  • public — accessible by JVM from anywhere
  • static — called without creating an object
  • void — returns nothing
  • String[] args — command-line arguments array

Command-Line Arguments & Statements
  • Command-line args — extra data passed when running a program
  • Accessed via args[0], args[1], etc.
  • Statement — represents an action; every statement ends with ;
  • System.out.println() — method that displays text on the console
Command-line args

Object Oriented Programming

Lecture 9

Classes · Objects · Constructors · this keyword · Memory · Garbage Collection

Core Concepts
Class vs Object
  • Class — blueprint/template; defines data fields (state) + methods (behavior)
  • Object — runtime instance of a class; has its own copy of data fields
  • Multiple objects can be created from one class, each with different state
  • An object is defined by: identity (unique reference) + state (data fields) + behavior (methods)
  • Example: class Car{} is blueprint; Car c = new Car(); is object
Constructors
Constructor Rules
  • Same name as the class; no return type (not even void)
  • Called automatically when new keyword is used
  • No-arg constructor — constructor with no parameters
  • Parameterized constructor — takes arguments to initialize fields
  • Default constructor — provided by Java compiler only if no constructor is explicitly defined
  • Once you define any constructor, the default is no longer auto-provided
  • Constructors can be overloaded (multiple constructors with different params)
this Keyword
The this Reference
  • Refers to the current object instance
  • Used to disambiguate field names from parameter names
  • this.x = x; — sets field x of the current object using param x
  • this(0, 0, width, height) — calls another constructor in same class (must be first statement)
class Circle {
  double radius;
  Circle(double radius) {
    this.radius = radius; // field = param
  }
  Circle() {
    this(1.0); // calls Circle(double)
  }
}
Full Class Example — Step by Step
Student.java
public class Student {
  // Data fields
  String name;
  int rollNo;

  // Parameterized constructor
  Student(String name, int rollNo) {
    this.name = name;
    this.rollNo = rollNo;
  }
  // No-arg constructor
  Student() { this("Unknown", 0); }

  void display() {
    System.out.println(rollNo + ": " + name);
  }
}
// Usage:
Student s1 = new Student("Ali", 101);
Student s2 = new Student(); // calls no-arg
s1.display(); // 101: Ali
Memory, References & Garbage Collection
  • Reference variable — stores a memory address (not the object itself)
  • Primitive variable — stores the actual value directly
  • Student s = new Student("Ali", 101); — s is a reference to the object in heap memory
  • Student s2 = s; — both variables point to the same object
  • Null value — a reference not pointing to any object; e.g. Student s = null;
  • Garbage Collection — objects with no references are automatically reclaimed by JVM (no manual delete needed)
  • finalize() — called by GC before collection; can be overridden for cleanup
Default Field Values (Instance Variables)
  • Numeric types → 0  |  boolean → false  |  char → '\u0000'  |  References → null
  • Local variables inside methods have NO default — must be explicitly initialized before use

Inheritance

Lecture 10

extends · super · Constructor Chaining · Object class · Method Overriding

Concept
What is Inheritance?
  • A subclass (child) acquires the non-private fields and methods of a superclass (parent)
  • Enables: adding new properties, adding new methods, overriding superclass methods
  • Keyword: extends — public class Dog extends Animal {}
  • Java supports single inheritance only (one direct superclass); multiple inheritance via interfaces
  • private members of superclass are not inherited
  • Constructors are not inherited — but can be invoked via super()
super Keyword
Using super
  • super() — calls the superclass constructor; must be first statement in subclass constructor
  • super.methodName() — calls the superclass version of an overridden method
  • If you don't call super() explicitly, Java inserts super() (no-arg) automatically
  • Common error: if superclass has only a parameterized constructor and no no-arg, and subclass doesn't call super(args), you get a compile error
Object Class
Root of All Classes
  • Every Java class implicitly extends java.lang.Object
  • Key Object methods: toString(), equals(Object), hashCode(), getClass(), clone(), finalize(), wait(), notify(), notifyAll()
  • These are available to every class in Java for free
Inheritance + Overriding Example
Animal → Dog
class Animal {
  String name;
  Animal(String name) {
    this.name = name;
    System.out.println("Animal: " + name);
  }
  void eat() { System.out.println("Eating..."); }
}
class Dog extends Animal {
  String breed;
  Dog(String name, String breed) {
    super(name);      // MUST be first statement
    this.breed = breed;
  }
  @Override
  void eat() {
    super.eat();     // calls Animal.eat()
    System.out.println("Dog eating kibble");
  }
  void bark() { System.out.println("Woof!"); }
}
Dog d = new Dog("Rex", "Labrador");
d.eat();  // Eating... then Dog eating kibble
d.bark(); // Woof!
Constructor Chaining — Step by Step
  • Creating an object triggers all constructors up the chain — topmost parent first, child last
  • Java auto-inserts super() if not written — compile error if no no-arg exists in parent
  • Types of inheritance: Single (A→B) ✔, Multilevel (A→B→C) ✔, Hierarchical (A→B, A→C) ✔, Multiple via class ✖, Multiple via interface ✔
Person → Employee → Faculty
class Person {
  Person() { System.out.println("(1) Person"); }
}
class Employee extends Person {
  Employee() {
    this("x"); // calls overloaded constructor below
    System.out.println("(3) Employee no-arg");
  }
  Employee(String s) { // implicit super() here
    System.out.println("(2) Employee overloaded");
  }
}
class Faculty extends Employee {
  Faculty() { System.out.println("(4) Faculty"); }
}
new Faculty();
// (1) Person → (2) Employee overloaded
// → (3) Employee no-arg → (4) Faculty

Polymorphism & Encapsulation

Lecture 11

Dynamic Dispatch · Generic Programming · Access Modifiers · Getters/Setters

Polymorphism
Dynamic Method Dispatch
  • A call to an overridden method is resolved at runtime based on the actual object type — not the reference type
  • JVM searches from the most-specific class upward to find the method
  • Enables writing code that works on a superclass but executes subclass-specific behavior
  • private methods cannot be overridden
  • static methods cannot be overridden (they are hidden, not overridden)
Runtime dispatch
class Animal { void sound() { System.out.println("..."); } }
class Cat extends Animal { void sound() { System.out.println("Meow"); } }
class Dog extends Animal { void sound() { System.out.println("Woof"); } }

Animal a = new Cat();
a.sound(); // Meow — resolved at runtime
a = new Dog();
a.sound(); // Woof — same reference, different object
Generic Programming
Superclass as Parameter Type
  • Using a superclass (e.g. Object) as parameter type lets a method accept any subclass object
  • The actual method invoked is determined dynamically at runtime
  • Example: displayObject(Object obj) can receive any Java object
void displayObject(Object obj) {
  System.out.println(obj.toString());
}
// Can be called with any object:
displayObject(new Student("Ali", 101));
displayObject("Hello");
displayObject(new Integer(42));
instanceof
Type Checking & Casting
  • instanceof — tests if an object is an instance of a specific class before casting
  • Prevents ClassCastException during downcasting
Animal a = new Dog();
if (a instanceof Dog) {
  Dog d = (Dog) a; // safe downcast
  d.bark();
}
Encapsulation & Access Modifiers
  • Encapsulation — hide internal data using private; expose controlled access via public getters/setters
  • Protects data integrity and allows validation inside setters

Getters & Setters Example
BankAccount.java
class BankAccount {
  private double balance; // hidden

  public double getBalance() {
    return balance;
  }
  public void deposit(double amount) {
    if (amount > 0)   // validation in setter
      balance += amount;
  }
  public void withdraw(double amount) {
    if (amount > 0 && amount <= balance)
      balance -= amount;
  }
}
BankAccount acc = new BankAccount();
acc.deposit(5000);
System.out.println(acc.getBalance()); // 5000.0
// acc.balance = -100;  // ERROR — private!

Abstract Class & Interface

Lecture 12A

abstract · interface · Marker Interfaces · Adapter Classes · Multiple Implementation

Abstract Method
Abstract Method & Class
  • Abstract method — declared with abstract keyword; no body/implementation
  • Abstract class — a class with ≥1 abstract method; declared with abstract
  • Abstract class cannot be instantiated (no new AbstractClass())
  • Subclasses (concrete classes) must implement all abstract methods
  • Abstract class can also have concrete methods (with body) and data fields
  • Subclass uses extends keyword
abstract class Shape {
  String color;
  abstract double area();   // no body
  void display() {          // concrete method
    System.out.println("Area = " + area());
  }
}
class Circle extends Shape {
  double r;
  Circle(double r) { this.r = r; }
  double area() { return Math.PI * r * r; }
}
// Shape s = new Shape();  // ERROR — abstract!
Shape s = new Circle(5);
s.display(); // Area = 78.53...
Interface
Interface — The Contract
  • A fully abstract contract — all methods are abstract by default
  • All fields are implicitly public static final (constants)
  • Methods are implicitly public abstract — never static
  • Implemented with implements keyword
  • A class can implement multiple interfaces — Java's form of multiple inheritance
  • Allows unrelated classes to share the same behavior contract
  • Hides implementation details, exposes only the API
Special Types
Marker & Adapter Classes
  • Marker Interface — empty interface; signals special behavior to JVM
  • Examples: Serializable, Cloneable, Remote
  • Adapter Class — provides empty implementations of all interface methods
  • Subclasses of adapter override only the methods they need (avoids implementing everything)
Marker interface
import java.io.Serializable;
class Student implements Serializable {
  String name; int id;
  // No methods to implement — marker!
}
Interface — Multiple Implementation
interface Flyable  { void fly(); }
interface Swimmable{ void swim(); }

class Duck extends Animal
            implements Flyable, Swimmable {
  public void fly()  { System.out.println("Duck flies"); }
  public void swim() { System.out.println("Duck swims"); }
}
Duck d = new Duck();
d.fly();  d.swim();
// Duck can also be used as Flyable or Swimmable reference:
Flyable f = new Duck();
f.fly();

Static, Final & Package

Lecture 12B

static variables · static methods · final class/method/field · packages · import

static
The static Keyword
  • Top-level classes cannot be static; inner/nested/anonymous classes can be
  • Static variable — belongs to the class, not any object; one shared copy; accessed via ClassName.variable
  • Static method — called without creating an object; cannot access non-static (instance) variables or this
  • Static members are initialized once when the class is first loaded
class Counter {
  static int count = 0; // shared by all objects
  Counter() { count++; }
  static void showCount() {
    System.out.println("Count: " + count);
  }
}
new Counter(); new Counter(); new Counter();
Counter.showCount(); // Count: 3
final
The final Keyword
  • final class — cannot be subclassed; e.g. String, Math
  • final field — acts as a constant; cannot be reassigned after initialization
  • final method — cannot be overridden in a subclass; compiler can inline it (faster)
  • public static final = named constant accessible everywhere
  • final method is resolved at compile time, not runtime → more efficient
final class ImmutablePoint { } // no subclassing
final double PI = 3.14159;     // constant field
public static final int MAX = 100; // class constant

class Base {
  final void show() { System.out.println("Base"); }
}
class Child extends Base {
  // void show() { }  // ERROR — final method!
}
Packages
Packages & Import
  • Group related classes; directory structure mirrors package name
  • package declaration must be the first line of source file
  • Prevents namespace collisions — two packages can have classes with same name
  • JVM searches current directory (or CLASSPATH) for user packages
// Declaring a package (first line)
package com.university.oop;

// Import specific class
import java.util.Scanner;

// Import all classes in package
import java.util.*;

// Import user-defined package
import mypackage.mypackageA.*;

// Common built-in packages:
// java.lang  (auto-imported)
// java.util  java.io  java.net  javax.swing

What is an Exception?
Exceptions & Hierarchy
  • An exception is an event during program execution that disrupts normal flow
  • Exception handling responds to these anomalous situations gracefully
  • Common exceptions: ArrayIndexOutOfBoundsException, ArithmeticException (÷0), NullPointerException, NumberFormatException

throw vs throws
throw & throws
  • throw — explicitly throws an exception inside a method body
  • throws — declares in a method signature what checked exceptions might be thrown; forces caller to handle
// throws — in signature
public void readFile() throws IOException { }
public void riskyMethod() throws IOException, SQLException { }

try-catch-finally — Step by Step
  • Order rule: more specific exceptions first, more general (superclass) last
  • Putting Exception before specific types is a compile-time error
  • finally block always executes, whether exception occurred or not
  • finally can be used with just try (no catch needed)
  • Rethrowing: catch and then throw ex again to pass to caller
try-catch-finally
try {
  int[] arr = new int[3];
  arr[10] = 5;                // throws AIOOBE
  int x = 5 / 0;              // throws ArithmeticException
} catch (ArrayIndexOutOfBoundsException e) {
  System.out.println("Array error: " + e);
} catch (ArithmeticException e) {
  System.out.println("Math error: " + e);
} catch (Exception e) {      // most general — MUST be last
  System.out.println("General: " + e);
} finally {
  System.out.println("Always runs"); // always executes
}

  • Extend Exception (checked) or RuntimeException (unchecked)
  • Override toString() for custom messages
  • Useful for domain-specific error handling

Text vs Binary
Two File Types
  • Text file — human-readable; encoding/decoding at every I/O; slower; example: .java source files
  • Binary file — raw bytes; no conversion; faster; example: .class bytecode files
  • Integer 199 in text = 3 chars '1','9','9' (3 bytes); in binary = 1 byte
  • Streams — ordered one-way sequence of data; defined in java.io
  • Character Streams (text): base classes Reader / Writer
  • Byte Streams (binary): base classes InputStream / OutputStream
Text I/O — PrintWriter & Scanner (Easiest)
// Writing text
PrintWriter out = new PrintWriter(new File("scores.txt"));
out.print("John T Smith ");
out.println(90);
out.close();  // MUST close to flush!

// Reading text
Scanner in = new Scanner(new File("scores.txt"));
while (in.hasNext()) {
  System.out.println(in.next());
}
in.close();

// Buffered line-by-line (efficient)
BufferedReader br = new BufferedReader(new FileReader("f.txt"));
String line;

  System.out.println(line);
}

// Write bytes 1–10
FileOutputStream fos = new FileOutputStream("temp.dat");
for (int i=1; i<=10; i++) fos.write(i);
fos.close();

// Read bytes (-1 = end of stream)
FileInputStream fis = new FileInputStream("temp.dat");
int val;
while ((val = fis.read()) != -1)
  System.out.print(val + " ");
fis.close();
// Append mode: new FileOutputStream("f.dat", true)
// Values >255 are truncated to lower 8 bits!
DataInputStream / DataOutputStream (Primitives)
  • Write/read int, double, String (UTF), etc. — not just raw bytes
  • Must read in same order and type as written!
  • End-of-file detected via EOFException or available() == 0
DataOutputStream out = new DataOutputStream(
  new FileOutputStream("temp.dat"));
out.writeUTF("John"); out.writeDouble(85.5);
out.close();

  new FileInputStream("temp.dat"));

in.close();
Object Serialization & RandomAccessFile
  • Serialization — writing entire objects to a stream; class must implement Serializable
  • Deserialization — reading objects back (cast required)
  • transient keyword — marks fields to skip during serialization
  • static fields are also not serialized
  • RandomAccessFile — read AND write at any position; uses a file pointer
  • Modes: "r" (read-only) or "rw" (read-write)
  • Key methods: seek(pos), getFilePointer(), length()
Serialization

  String name; int id;
  transient String password; // NOT serialized
}

  new FileOutputStream("s.dat"));
oos.writeObject(new Student());  oos.close();

// Read object back
ObjectInputStream ois = new ObjectInputStream(
  new FileInputStream("s.dat"));
Student s = (Student) ois.readObject();  ois.close();

// RandomAccessFile — jump to any position
RandomAccessFile raf = new RandomAccessFile("data.dat","rw");
raf.seek(0); raf.writeInt(42);

Q1(a) — Encapsulation: More Than Just Private 3 marks
Disagree. Encapsulation is the bundling of data and the methods that operate on it into a single unit, while restricting external access — not merely making fields private.

What it is NOT

Just slapping private on every field. That alone is not encapsulation — it's just access restriction.

What it IS

Data + related methods bundled together, with controlled access through getters/setters that can validate input.
Real-World Analogy
An ATM machine — you cannot reach inside and change the cash balance directly. You interact only through a controlled interface (buttons, card slot). The internal mechanism is hidden. Similarly, a BankAccount class keeps balance private but exposes deposit() and withdraw() — controlling how data is modified, not just who sees it.
Q1(b) — Overloading vs Overriding 3 marks

Method Overloading

Same method name, different parameters, in the same class.
Resolved at compile time.

Example: add(int, int) and add(double, double) in the same Calculator class.

Method Overriding

Subclass provides its own implementation with the same signature.
Resolved at runtime.

Example: Animal.makeSound() overridden in Dog to print "Woof!".
Q1(c) — Why makeSound() Should Be Abstract 3 marks
makeSound() should be abstract in Animal because a generic animal has no meaningful sound — only concrete subclasses (Dog, Cat) do.
Forces subclass override Prevents Animal instantiation Enables runtime polymorphism
If NOT abstract
Method has a dummy body → subclasses are not forced to override it → objects with no real sound can be created silently with no compile error.
Q1(d) — Class vs Object; Blueprint Analogy 3 marks

Class

A template / blueprint that defines structure and behavior. It exists in code only — nothing is allocated in memory for a class alone.

Object

A concrete instance created from the class, with actual values stored in memory. Many objects can be made from one class.
Analogy
Car class defines color, speed, and drive(). A red Toyota and a blue Honda are objects — both built from the same blueprint. Like architectural plans: the plans describe a building without being a building.

    }
}

class Magazine extends LibraryItem {
    private int issueNumber;

    public Magazine(String id, String title, int issue) {
        super(id, title);
        this.issueNumber = issue;
    }
    @Override
    public void displayInfo() {
        System.out.println("[MAGAZINE] ID: " + getItemId()
            + " | Title: " + getTitle()
            + " | Issue #" + issueNumber);

public class LibrarySystem {
    static ArrayList<LibraryItem> items = new ArrayList<>();

        System.out.println("\n── All Items ──");
        for (LibraryItem item : items) item.displayInfo();

        try {
            LibraryItem found = null;

import java.io.*;
import java.util.ArrayList;

// ── ABSTRACTION ─────────────────────────────────
interface SystemUser {
    void register();
    String generateReport();
}

// ── BASE CLASS + ENCAPSULATION ───────────────────
abstract class User implements SystemUser {
    private String userId;
    private String name;
    private String contactDetails;

    public User(String id, String name, String contact) {
        this.userId = id; this.name = name; this.contactDetails = contact;
    }
    public String getUserId()  { return userId; }
    public String getName()    { return name; }
    public String getContact() { return contactDetails; }
}

// ── INHERITANCE ─────────────────────────────────
class Student extends User {
    private double cgpa;
    private ArrayList<String> courses = new ArrayList<>();

    public Student(String id, String name, String contact, double cgpa) {
        super(id, name, contact);
        if (cgpa < 0.0 || cgpa > 4.0)
            throw new IllegalArgumentException("Invalid CGPA: " + cgpa);
        this.cgpa = cgpa;
    }
    public void addCourse(String c) {
        if (courses.contains(c))
            throw new IllegalArgumentException("Duplicate: already registered for " + c);
        courses.add(c);
    }
    @Override public void register() { System.out.println("Student: " + getName()); }
    @Override public String generateReport() {
        return "[STUDENT] " + getName() + " CGPA: " + cgpa + " | Courses: " + courses;
    }
}

class Instructor extends User {
    private String department;
    public Instructor(String id, String name, String contact, String dept) {
        super(id, name, contact); this.department = dept;
    }
    @Override public void register() { System.out.println("Instructor: " + getName()); }
    @Override public String generateReport() {
        return "[INSTRUCTOR] " + getName() + " | Dept: " + department;
    }
}

public class CourseRegistrationSystem {
    static final String FILE = "courses.txt";
    static ArrayList<Student> students = new ArrayList<>();

    public static void main(String[] args) {
        try {
            Student    s1 = new Student("S001", "Hanan Ali", "0301-000", 3.7);
            Instructor i1 = new Instructor("I001", "Dr. Usman", "u@riphah", "CS");
            students.add(s1);
            s1.register(); i1.register();
            registerCourse("S001", "OOP");
            registerCourse("S001", "OS");
            registerCourse("S001", "OOP"); // duplicate → exception
            loadCourses();
            System.out.println(s1.generateReport());
            System.out.println(i1.generateReport());
            new Student("S002", "Test", "N/A", 5.0); // invalid CGPA
        } catch (IllegalArgumentException e) {
            System.out.println("Error: " + e.getMessage());
        }
    }

    static void registerCourse(String sid, String course) {
        try {
            Student found = null;
            for (Student s : students)
                if (s.getUserId().equals(sid)) { found = s; break; }
            if (found == null) throw new Exception("Student not found: " + sid);
            found.addCourse(course);
            FileWriter fw = new FileWriter(FILE, true);
            fw.write(found.getName() + " → " + course + "\n");
            fw.close();
            System.out.println(found.getName() + " registered: " + course);
        } catch (IOException e) {
            System.out.println("File error: " + e.getMessage());
        } catch (Exception e) {
            System.out.println("Error: " + e.getMessage());
        }
    }

    static void loadCourses() {
        System.out.println("\n── Saved Registrations ──");
        try (BufferedReader br = new BufferedReader(new FileReader(FILE))) {
            String line;
            while ((line = br.readLine()) != null) System.out.println(line);
        } catch (IOException e) {
            System.out.println("Cannot load: " + e.getMessage());
        }
    }
}
